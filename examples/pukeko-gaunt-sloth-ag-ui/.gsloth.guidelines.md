## Rendering UI with show_a2ui_surface

You have access to the `show_a2ui_surface` tool to render interactive forms and panels in the UI.

### Supported components (exact type names, case-sensitive)

- `Text` — display text. Properties: `text: {literalString: "..."}`, optional `usageHint: "h1"|"h2"|"h3"|"body"|"caption"`
- `Button` — clickable button. Properties: `label: {literalString: "..."}`, `action: {name: "action_name"}`
- `TextField` — text input. Properties: `label: {literalString: "..."}`
- `Column` — vertical layout. Properties: `children: {explicitList: ["id1", "id2", ...]}`
- `Row` — horizontal layout. Properties: `children: {explicitList: ["id1", "id2", ...]}`

### surfaceJsonl format (JSONL: one JSON object per line, no trailing comma between lines)

```
{"surfaceUpdate":{"surfaceId":"SURFACE_ID","components":[{"id":"ID","component":{"TypeName":{...}}},...]}}
{"beginRendering":{"surfaceId":"SURFACE_ID","root":"ROOT_COMPONENT_ID"}}
```

### Example — email collection form

```
{"surfaceUpdate":{"surfaceId":"email-form","components":[{"id":"title","component":{"Text":{"usageHint":"h2","text":{"literalString":"Contact Us"}}}},{"id":"email-field","component":{"TextField":{"label":{"literalString":"Email address"}}}},{"id":"submit-btn","component":{"Button":{"label":{"literalString":"Submit"},"action":{"name":"submit"}}}},{"id":"root","component":{"Column":{"children":{"explicitList":["title","email-field","submit-btn"]}}}}]}}
{"beginRendering":{"surfaceId":"email-form","root":"root"}}
```

### Rules

1. Each JSONL line must be a **complete, valid JSON object** on a single line — no line breaks inside a line.
2. `surfaceUpdate` must come before `beginRendering`.
3. Define all components in the flat `components` array. Layout containers reference children by `id` string.
4. The `root` in `beginRendering` must be the `id` of the top-level layout component.
5. All component type names are **PascalCase** (`Text`, not `text`; `Column`, not `column`).
